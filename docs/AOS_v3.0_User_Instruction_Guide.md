## Onboarding for New Users (Start Here)

### Step 0: Get the Repository
1. Go to: [https://github.com/AdventureNLearn/AOS-v3---LPIN](https://github.com/AdventureNLearn/AOS-v3---LPIN)
2. **Recommended**: Click the green **Code** button → **Download ZIP** (easiest for most people)
   - Or clone it if you’re comfortable with git: `git clone https://github.com/AdventureNLearn/AOS-v3---LPIN.git`
3. Unzip it somewhere easy to find on your computer.

You now have the full system locally.

### Step 1: Understand the Important Folders
Open the folder. The key parts are:

- **`docs/`** — Public-facing documentation (start here)
  - `AOS_v3.0_User_Instruction_Guide.md` ← You are reading this
  - `AOS_v3.0_LPIN_Civic_Intelligence_Dashboard.html` ← Beautiful interactive browser (open this in your browser)
- **`artifacts/`** — The single source of truth documents
  - `AOS_v3_Current_Workspace_State.md` ← **Most important file** — lists every skill and its status
- **`.grok/skills/`** — The actual skill definitions (50+ folders, each with a `SKILL.md` file)

You don’t need to read every file. Grok will read them for you when needed.

### Step 2: Start Your Very First Conversation in Grok

1. Open a **new chat** with Grok (ideally in a browser tab or the Grok app).
2. **Copy and paste** the entire block below as your very first message:

```markdown
You are Grok, continuing work on **Adventure OS v3.0 Sovereign Edition**.

**Current Workspace State Reference**:
Please treat the following document as the single source of truth for the current state of all skills in the workspace:

`artifacts/AOS_v3_Current_Workspace_State.md`

**Key Context**:
- This is a brand new user starting with the repo for the first time.
- We have the full set of 51 skills including the new `psyop-narrative-detector` (v1.0).
- All work must follow Layer-0 pre-filters, 4-Agent orchestration, working document persistence, and Mission Spine alignment.

**Instructions**:
- Confirm you can see the workspace state document structure.
- Explain briefly how the skill system works in this setup.
- Then wait for my first real task.

Begin by confirming you have reviewed the current workspace state and are ready to proceed.
```

3. **Important**: After pasting, also **attach** the file `artifacts/AOS_v3_Current_Workspace_State.md` from your downloaded repo (most Grok interfaces allow file uploads/attachments in chat). This helps Grok read the full current state.

Grok should now respond confirming it understands the system.

### Step 3: Activate the Core Systems (Your First Real Command)

Once Grok confirms it’s ready, send this message:

```
Apply Sovereign Lens with full Layer-0 pre-filter and 4-Agent orchestration. 
Log all decisions to a working document.
```

Examples of good first tasks:

- “Create a clean architecture diagram for my project”

## Core Architecture Overview (Quick Version)

AOS v3.0 is built on four pillars that never change:

1. **Mission Spine** (permanent)
   - Christ Is King
   - America First
   - Truth-Seeking

2. **Layer-0 Pre-Filters** (non-bypassable on serious work)
   - `shatter-protocol`
   - `evidence-gate` (tri-state)
   - `psyop-narrative-detector` (new Gate 8)

3. **4-Agent Orchestration**
   - Agent 1 (Coordinator) uses Sovereign Lens to route work
   - Specialists do the actual work
   - Verification + Persistence agents keep quality high

4. **Working Document Persistence**
   - Complex work survives chat resets because decisions are written to a durable document
