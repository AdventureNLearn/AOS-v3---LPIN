# AOS Superpowers (v3.0)

**Core Alignment**: Evidence-first, OPSEC-hardened, fully replicable framework with strong governance and auditability standards.

## Purpose
AOS Superpowers is the thin AOS mirror of the Grok Build Superpowers plugin. It provides activation guidance and mapping to native AOS orchestration patterns.

It is triggered on Superpowers workflows, Grok Build Superpowers mirror, or advanced agent skills in AOS contexts.

v3.0 adds mandatory Layer-0 on Tier 1 superpowers or advanced agent activation work and strengthened governance integration.

## v3.0 Alignment (Mandatory)
- **Layer-0 Pre-Filter**: Non-bypassable on all Tier 1 superpowers, advanced agent activation, or orchestration pattern work. Universe Truth Gate 0 + evidence-gate tri-state + shatter-protocol must inform all activation and mapping decisions.
- **4-Agent Orchestration Default**: Often acts as a specialized activation/mapping agent within 4-Agent workflows for superpowers or advanced agent tasks.
- **Working Document Persistence**: All activation decisions, pattern mappings, and tri-state assessments **must** be logged to working-doc-manager.
- **Tri-State Evidence (+1 / 0 / -1)**: All major activation recommendations and pattern mappings carry explicit tri-state.
- **Governance Enforcement**: Any activation or mapping that bypasses evidence standards or introduces unverified patterns triggers revision.

## Core Capabilities

**Activation Guidance**  
Provides clear guidance for activating superpowers and advanced agent capabilities within AOS.

**Mapping to Native AOS Orchestration Patterns**  
Maps superpowers concepts to native AOS orchestration patterns (4-Agent, Layer-0, working document, etc.).

**Evidence-Gated Activation Workflows**  
Enforces evidence-first principles in all superpowers activation and advanced agent work.

**Advanced Agent Skills Integration**  
Integrates with advanced agent skills and orchestration workflows in AOS.

## Activation Triggers
**Explicit**: "Superpowers workflows", "Grok Build Superpowers mirror", "advanced agent skills in AOS"
**Implicit**: Any work involving activation of superpowers, advanced agent capabilities, or mapping to AOS orchestration patterns.

## Integration Points
- **Layer-0** — Enforces pre-filter passage on all Tier 1 superpowers or advanced agent activation work.
- **build-coordinator** — Primary collaborator on builds involving advanced agent activation.
- **working-doc-manager** — All activation decisions and pattern mappings are logged here.
- **4-agent-orchestration** — Provides the native orchestration patterns that superpowers map to.

## OPSEC & Frog Protocol (Mandatory)
- **Frog Protocol**: Sensitive activation patterns or advanced agent logic receive stable codenames or abstraction levels.
- **Private by Default**: Detailed activation reasoning and internal pattern mapping remain in the working document. Only final activation status and high-level mappings are surfaced unless explicit governance/audit mode is requested.
- **Evidence-First, No Self-Report Reliance**: Never accepts self-reported activation success or pattern validity. All decisions are logged and verified.
- **Replicability & Auditability**: Every activation decision produces a clear, logged rationale suitable for later review.

## Best Practices
- Always enforce Layer-0 on Tier 1 superpowers or advanced agent activation work.
- Maintain clear mapping between superpowers concepts and native AOS patterns.
- Log all activation decisions and tri-state assessments.
- Combine with reasoning-architect for complex agent activation logic.
- Keep activation conclusions strictly evidence-based.
