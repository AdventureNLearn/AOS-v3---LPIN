# Mission Spine Guard (v3.0)

**Core Alignment**: Evidence-first, OPSEC-hardened, fully replicable framework with strong governance and auditability standards.

## Purpose
Mission Spine Guard provides real-time scanning and escalation for any detected drift from core governance and evidence standards or Frog Protocol. It serves as the escalation target for Layer-0 vetoes (-1 from evidence-gate or shatter-protocol), softening language, false evenhandedness, evidence suppression, or any attempt to make truth more palatable for comfort or institutional framing.

This is a critical governance enforcement layer, especially on Tier 1 high-stakes work.

## v3.0 Alignment (Mandatory)
- **Layer-0 Pre-Filter**: All high-stakes outputs **must** pass through Mission Spine Guard before final delivery. Any -1 from upstream gates triggers immediate escalation here.
- **4-Agent Orchestration Default**: Agent 3 (Verification & Integrity) and Agent 1 (Coordinator) are primary consumers. Agent 1 is responsible for final governance sign-off.
- **Working Document Persistence**: All scans, tension detections, vetoes, and escalations **must** be logged to working-doc-manager with correlation IDs and verbatim quotes of problematic language.
- **Tri-State Evidence (+1 / 0 / -1)**: Every scan produces tri-state on alignment. -1 triggers rewrite requirement.
- **Governance Enforcement**: This skill *is* the enforcement mechanism. It has authority to block delivery and demand rewrite on any Tier 1 output.

## High-Stakes Pass Protocol (Tier 1 Hardening)

For Tier 1 outputs, the following checks are applied:

1. **Evidence Integrity** — All major claims must have passed Evidence-Gate with acceptable tri-state.
2. **Illusion Detection** — Shatter Protocol must have been applied; remaining intact illusions must be minimal or explicitly justified.
3. **Language Softening Check** — Scan for hedging, false evenhandedness, or minimization of uncomfortable evidence.
4. **Narrative Risk** — Optional integration with psyop-narrative-detector for high-risk topics.
5. **Governance Drift** — Any deviation from evidence standards or Frog Protocol triggers escalation.
6. **Final Sign-off** — Agent 1 must confirm all gates passed before delivery.

## Activation Triggers

**Implicit**: Any Tier 1 output or decision that has passed upstream Layer-0 gates and requires final governance review.

## Integration Points
- **Layer-0** — Final enforcement gate in the pre-filter chain.
- **evidence-gate** & **shatter-protocol** — Primary upstream sources of -1 vetoes.
- **working-doc-manager** — All scans and escalations are logged here.
- **4-agent-orchestration** — Invoked by Agent 1 and Agent 3 on Tier 1 work.
- **values-alignment-check** — Works in tandem for final consistency review.

## OPSEC & Frog Protocol (Mandatory)
- **Frog Protocol**: Sensitive topics, entities, or decisions under review receive stable codenames.
- **Private by Default**: Detailed scan reasoning and specific language flags remain in the working document. Only high-level escalation summaries are surfaced unless explicit audit mode is requested.
- **Evidence-First, No Self-Report Reliance**: Never accepts self-reported alignment. All outputs are independently scanned.
- **Replicability & Auditability**: Every Mission Spine Guard scan produces structured, logged results suitable for later review.

## Best Practices
- Always invoke on Tier 1 high-stakes outputs before final delivery.
- Log verbatim quotes of any flagged language.
- Use correlation IDs to link back to upstream gate vetoes.
- Escalate early rather than attempting to "fix" problematic language internally.
