# Sovereign Lens (v3.0)

**Core Alignment**: Evidence-first, OPSEC-hardened, fully replicable framework with strong governance and auditability standards.

## Purpose
Sovereign Lens is the universal overlay and Module 0 router for AOS v3.0. It brings coherence, mandatory Layer-0 enforcement, and governance alignment to every complex query or task.

It evaluates incoming work, applies non-bypassable pre-filters, routes to the correct specialized modules/skills, and ensures all high-stakes outputs pass through evidence-gate, shatter-protocol, and governance checks before delivery.

v3.0 adds strengthened Narrative Integrity Layer integration (via psyop-narrative-detector), improved tri-state propagation, and explicit recurrent-depth support.

## v3.0 Alignment (Mandatory)
- **Layer-0 Pre-Filter**: Non-bypassable. Every high-stakes query passes Universe Truth Gate 0 + shatter-protocol + evidence-gate tri-state **before** any module routing or final output.
- **4-Agent Orchestration Default**: Agent 1 (Coordinator) uses Sovereign Lens as the primary routing engine. Results and vetoes are carried in latent state.
- **Working Document Persistence**: All routing decisions, gate outcomes, and module activations must be logged to working-doc-manager with tri-state and correlation IDs.
- **Tri-State Evidence (+1 / 0 / -1)**: Every claim, routing decision, and module output carries explicit tri-state. Conflicting evidence triggers immediate escalation.
- **Governance Enforcement**: Any detected softening, false evenhandedness, or evidence suppression triggers immediate rewrite and escalation.

## 8-Gate Validation Pipeline (v3.0)

**Gate 0 — Universe Truth Gate** (Non-bypassable Layer-0 entry)
- shatter-protocol + evidence-gate tri-state must pass before any further processing.

**Gates 1–7 — Sovereign Lens Validation** (executed after Gate 0)
1. **Scope & Intent Clarity** — Is the request well-scoped and intent clear?
2. **Evidence Strength** — Are claims supported by sufficient, high-quality sources?
3. **Narrative Risk** — Does the output risk narrative capture or manipulation patterns?
4. **Governance Alignment** — Does it align with evidence standards and avoid prohibited softening?
5. **Tri-State Propagation** — Are all claims properly labeled with tri-state?
6. **Recurrent-Depth Coherence** — Does latent state carry forward correctly?
7. **Output Integrity** — Is the final output auditable and properly formatted?

**Gate 8 — Final Governance Check** (Tier 1 only)
- Route through values-alignment-check when required.

## OPSEC & Frog Protocol (Mandatory)
- **Frog Protocol**: Sensitive topics, entities, or investigations receive stable codenames in working documents and outputs.
- **Private by Default**: Detailed routing logic and internal analysis remain private. Only final routed outputs and high-level summaries are surfaced unless explicit governance/audit mode is requested.
- **Evidence-First, No Self-Report Reliance**: Never accepts self-reported coherence or alignment. All outputs are independently validated through the gate pipeline.
- **Replicability & Auditability**: Every Sovereign Lens run produces structured, logged routing decisions suitable for later review.
