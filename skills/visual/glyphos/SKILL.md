# Glyphos (v3.0)

**Core Alignment**: Evidence-first, OPSEC-hardened, fully replicable framework with strong governance and auditability standards.

## Purpose
Glyphos is used for designing glyph and operator systems in visualization work. It is activated when building layered symbolic encodings, defining visual operators, or creating structured multi-layer visual logic. It supports advanced work in visual-systems-architect.

v3.0 adds mandatory Layer-0 on Tier 1 symbolic/visual logic work and strengthened governance integration.

## v3.0 Alignment (Mandatory)
- **Layer-0 Pre-Filter**: Non-bypassable on all Tier 1 symbolic encoding, visual operator, or multi-layer visual logic work. Universe Truth Gate 0 + evidence-gate tri-state must inform all glyph and operator design decisions.
- **4-Agent Orchestration Default**: Often acts as a specialized visual logic agent within 4-Agent workflows for complex visualization or symbolic reasoning deliverables.
- **Working Document Persistence**: All glyph/operator design decisions, encoding rationale, and tri-state assessments **must** be logged to working-doc-manager.
- **Tri-State Evidence (+1 / 0 / -1)**: Glyph and operator systems should clearly support representation of tri-state evidence and decision states where applicable.
- **Governance Enforcement**: Any symbolic encoding that obscures evidence strength, hides veto paths, or misrepresents governance logic triggers revision.

## Core Capabilities

**Layered Symbolic Encodings**  
Designs multi-layer systems of glyphs and symbols for complex information representation.

**Visual Operator Definition**  
Defines reusable visual operators (transformations, filters, aggregations) that can be applied across visual systems.

**Structured Multi-Layer Visual Logic**  
Creates coherent visual logic systems that support reasoning, comparison, and governance workflows.

**Integration with Visual Systems Architect**  
Provides advanced symbolic and operator foundations that visual-systems-architect can build upon for complex deliverables.

## Activation Triggers
**Explicit**: "glyphos", "design glyph system", "visual operators", "symbolic encoding", "multi-layer visual logic"
**Implicit**: Any advanced visualization work requiring custom symbolic systems, operators, or layered visual logic beyond standard charts and diagrams.

## Integration Points
- **Layer-0** — Enforces pre-filter passage on all Tier 1 symbolic/visual logic work.
- **visual-systems-architect** — Provides advanced symbolic foundations for this role.
- **working-doc-manager** — All glyph/operator design decisions and encoding rationale are logged here.
- **reasoning-architect** — Supports symbolic representation of complex reasoning structures.
- **sovereign-lens** — Can inform prioritization of visual logic systems for high-stakes work.

## OPSEC & Frog Protocol (Mandatory)
- **Frog Protocol**: Sensitive processes or entities represented through custom glyphs receive stable codenames or abstraction levels.
- **Private by Default**: Detailed glyph/operator design rationale and internal iterations remain in the working document. Only final visual logic systems are surfaced unless explicit governance/audit mode is requested.
- **Evidence-First, No Self-Report Reliance**: Never accepts self-reported clarity or completeness of a symbolic system. All designs are reviewed for accurate representation of evidence and governance.
- **Replicability & Auditability**: Every glyph/operator system includes clear documentation of its logic and evidence mapping.

## Best Practices
- Always enforce Layer-0 on Tier 1 symbolic/visual logic work.
- Design glyphs and operators to clearly support tri-state evidence and veto path representation.
- Maintain clear documentation of encoding rationale and operator behavior.
- Iterate closely with visual-systems-architect on complex systems.
- Log all major design decisions and evidence mappings.
